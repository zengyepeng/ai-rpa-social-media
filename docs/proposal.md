# 毕业设计开题报告

**题目：基于AI Agent与影刀RPA的小红书智能内容运营系统**

## 一、选题背景与意义

### 1.1 背景
小红书已成为品牌营销的重要阵地，但内容运营面临三大痛点：选题靠人工经验、发布需重复操作、数据反馈滞后。传统RPA只能按固定流程执行，无法根据内容质量和平台规则动态调整策略。

### 1.2 意义
本项目将AI Agent作为“大脑”驱动RPA执行，解决以下问题：
- 自动抓取热门话题，AI生成笔记文案
- RPA自动完成发布、评论、数据采集
- 根据反馈数据调整内容策略

**实际价值**：个人博主可节省每天2小时重复操作，企业运营可降低50%人力成本。

## 二、国内外研究现状

### 2.1 RPA应用现状
影刀RPA在国内电商、新媒体领域应用广泛，但多数停留在固定流程自动化（如定时发布、批量点赞），缺乏智能决策能力。

### 2.2 AI Agent发展
OpenClaw等Agent框架能调用LLM完成复杂任务，但鲜有与RPA结合的落地案例。DeepSeek API成本低、响应快，适合中小规模应用。

### 2.3 现有方案不足
- 纯RPA：无法处理内容质量判断、话题选择等非结构化任务
- 纯AI Agent：无法操作小红书Web端（反爬严格）
- 两者结合：目前无成熟开源方案

**本项目定位**：填补AI+RPA在小红书运营场景的落地空白。

## 三、研究目标与内容

### 3.1 总体目标
构建一套AI Agent决策+RPA执行的智能运营系统，实现“选题-创作-发布-反馈”全链路自动化。

### 3.2 具体内容
1. **AI Agent模块**（香港服务器部署）
   - 基于OpenClaw框架，调用DeepSeek API
   - 功能：热门话题分析、文案生成、评论回复生成、数据复盘
   
2. **影刀RPA模块**（本地Windows运行）
   - 登录、发布笔记、评论互动、数据采集
   - 与AI Agent通过HTTP接口通信

3. **数据闭环**：RPA采集数据→AI分析→优化下一轮内容

### 3.3 技术栈
| 模块 | 技术选型 | 说明 |
|------|---------|------|
| AI Agent | OpenClaw + DeepSeek API | 成本低，支持多Agent协作 |
| 服务器 | 腾讯云香港轻量（2核4G） | 规避国内API限制 |
| RPA | 影刀RPA 6.0 | 社区版免费，支持Python脚本 |
| 数据存储 | SQLite + CSV | 轻量易维护 |

## 四、研究方法与技术路线

### 4.1 研究方法
- **原型法**：先跑通核心流程，再逐步优化
- **实验验证**：对比人工运营与系统运营的效率、内容质量

### 4.2 技术路线

```
用户指令 → AI Agent（香港服务器）
                ↓
         [任务分解]
    ┌───────────┼───────────┐
    ↓           ↓           ↓
 选题分析   文案生成   数据复盘
    ↓           ↓           ↓
    └───────────┼───────────┘
                ↓
          HTTP接口调用
                ↓
         影刀RPA（本地）
                ↓
    ┌───────────┼───────────┐
    ↓           ↓           ↓
 登录发布   评论互动   数据采集
                ↓
          ┌────┴────┐
          ↓         ↓
       小红书面板  本地数据库
```

### 4.3 关键实现细节
- AI Agent与RPA通过本地HTTP服务通信（端口8888）
- RPA每30分钟轮询一次任务队列
- 使用影刀的Python脚本功能实现JSON解析

## 五、预期成果与创新点

### 5.1 预期成果
1. 可运行的智能运营系统一套（含AI Agent服务端+RPA脚本）
2. 使用手册一份（含部署步骤、异常处理）
3. 测试数据：连续运行7天，对比人工运营效率

### 5.2 创新点
1. **架构创新**：AI Agent作为RPA的“决策层”，RPA作为“执行层”
2. **应用创新**：首次将OpenClaw+DeepSeek用于小红书内容运营
3. **成本优势**：总部署成本低于200元/月（香港服务器80元+API费用）

## 七、可行性分析

### 7.1 技术可行性
- **AI Agent**：OpenClaw官方文档完善，提供Python SDK，专科生可上手
- **RPA**：影刀RPA有社区版，提供免费课程和模板
- **接口通信**：HTTP+JSON方案成熟，影刀支持Python脚本调用requests库
- **风险点**：小红书反爬策略变化 → 方案：RPA模拟真人操作（随机延时、模拟鼠标轨迹）

### 7.2 资源可行性
- **硬件**：现有Windows笔记本即可运行RPA
- **资金**：香港服务器80元/月，DeepSeek API按量付费，预估月均50元
- **时间**：每天2小时，8周可完成

### 7.3 数据可行性
- 测试账号：用个人小红书账号（非认证）
- 数据量：每天发布3-5篇笔记，采集50条评论，足够验证效果

### 7.4 风险与应对
| 风险 | 概率 | 应对措施 |
|------|------|---------|
| DeepSeek API不稳定 | 低 | 增加重试机制，备选豆包API |
| 小红书账号被封 | 中 | 使用小号测试，控制操作频率 |
| 影刀RPA更新兼容性 | 低 | 锁定影刀版本6.0 |

## 结语

本项目立足专科层次的技术能力，选择成熟开源框架+国产RPA工具，聚焦“AI+RPA”在小红书运营场景的落地。技术栈可控、成本低、周期短，具备完整性和可行性。

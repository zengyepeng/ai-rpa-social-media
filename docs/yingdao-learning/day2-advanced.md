# 影刀RPA 系统性学习 — 第2天：进阶实战篇

## 一、元素捕获（核心技能）

### 4种捕获模式
| 模式 | 适用场景 | 说明 |
|------|---------|------|
| **捕获新元素** | 通用 | 标准捕获，鼠标移动到目标上蓝色高亮→左键点击 |
| **深度模式捕获** | Windows桌面软件 | 针对Win软件设计的捕获模式 |
| **CV智能模式捕获** | 企业微信、旺店通等特殊软件 | 图像识别方式捕获，注意受分辨率影响 |
| **AI辅助定位(v6.0)** | 动态变化的元素 | 图像+语义描述定位，兜底方案 |

### 元素定位原理
- 影刀元素定位本质是**XPath + 属性匹配**
- 稳定定位顺序：`id > 锚点 > innertext > class > index`

### 元素定位技巧
1. **默认选择器**：勾选节点属性定位
2. **XPath**：`//button[text()="提交"]`、`//div[@id="title"]`
3. **锚点**：用附近不变的元素做基准
4. **AI优化元素(v5.28)**：自动生成更稳定的XPath路径
5. **校验元素**：捕获后校验是否定位成功
6. **修复元素**：定位失败时，AI自动修复元素规则

### 常见元素问题解决
- **找不到元素** → 加等待/调整XPath/换锚点/AI修复
- **找到多个元素** → 缩小匹配范围/加锚点
- **隐藏元素** → Ctrl+Shift+左键单击，或设置元素属性
- **捕获偏移** → 分辨率/缩放问题，重启影刀

## 二、文件上传（小红书发布关键点）

### 方案一：拖拽上传（推荐优先）
```
1. 获取上传区域的元素坐标
2. 使用「拖拽文件到元素」指令
3. 传入本地文件路径
```

### 方案二：Windows系统对话框操作
```
1. 点击上传按钮 → 触发系统文件对话框
2. 切换到Windows桌面应用模块
3. 定位地址栏，输入文件完整路径
4. 按回车确认
5. 点击「打开」按钮
```

### 方案三：JS直接设置input值
```javascript
// 在「执行JS脚本」指令中使用
document.querySelector('input[type="file"]').value = 'C:\\path\\to\\image.jpg';
// 注意：浏览器安全限制，成功率可能不高
```

## 三、表单填写实战指南

### 通用表单填写流程
```
1. 点击输入框（获取焦点）
2. Ctrl+A全选 + Delete清空（避免追加）
3. 输入新内容
4. 如清空后输入 → 影刀有「清空并输入」组合指令
```

### 下拉框
- 原生select → 用「设置下拉框值」指令
- div模拟下拉 → 模拟点击展开→等待选项出现→点击选项
- 可搜索下拉 → 输入关键词→等待过滤→点击匹配项

### 提交前验证
```
1. 检查所有必填项是否有值
2. 截图保存当前表单状态
3. 点击提交
4. 等待成功/失败提示
5. 记录结果到日志或Excel
```

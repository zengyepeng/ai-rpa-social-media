## 案例一：码农飞哥 — 小红书爆款笔记抓取

**来源**：腾讯云开发者社区（2024-04）
**地址**：https://cloud.tencent.com/developer/article/2413974

### 业务场景
在小红书推荐页批量采集笔记 → 按点赞数筛选爆款(>1000) → 下载主图 → 后续仿写+发布

### 完整流程
```
1. 新建应用（PC自动化）
2. 新建【打开选择文件夹对话框】→ 让用户自定义图片保存目录
3. 【打开网页】→ 小红书推荐页
   URL: https://www.xiaohongshu.com/explore?channel_id=homefeed_recommend
4. 【批量抓取数据】指令 → 抓取列表页数据，保存至内置数据表格
   - 可加翻页功能实现多页抓取
5. 【循环数据表格内容】→ 遍历抓取结果
   循环内：
   a. 读取第F列 → 获取点赞数
   b. 【IF多条件】判断：点赞数 > 1000 → 认定为爆款
   c. 读取第A列 → 获取主图链接
   d. 【下载文件】→ 下载主图到指定目录
6. 循环结束 → 输出日志
```

### 关键指令
- `打开选择文件夹对话框` — 界面交互类
- `批量抓取数据` — 网页自动化，自动抓取列表页结构化数据
- `循环数据表格内容` — 流程控制，遍历批量数据
- `IF多条件` — 条件判断筛选

## 案例二：狗哥（Duck）— 完整图文笔记采集

**来源**：火山引擎开发者社区（2025-08）
**地址**：https://developer.volcengine.com/articles/7540136072844574730

### 业务场景
输入单个小红书笔记链接 → 自动采集标题/正文/标签/图片/互动数据 → 保存到Excel

### 完整流程
```
1. 【自定义对话框】→ 输入笔记链接 + 保存路径
2. 【打开网页】→ 打开笔记链接
3. 获取笔记内容：
   - 【获取元素对象(web)】→ 捕获标题区域 → 文本
   - 【获取元素对象(web)】→ 捕获正文区域 → 文本
   - 【获取相似元素列表(web)】→ 批量获取标签 → 循环拼接成tags文本
   - 【获取元素对象(web)】→ 点赞数/收藏数/评论数
4. 【创建文件夹】→ 以帖子标题命名
5. 下载图片（核心难点）：
   a. 【获取相似元素列表(web)】→ XPath定位图片
      XPath: //div[contains(@class, 'swiper-slide') 
             and (not(contains(@class, 'swiper-slide-duplicate-')) 
             or contains(@class, 'swiper-slide-active'))]//img
   b. 【ForEach列表循环】→ 遍历图片列表
   c. 设置变量 → 重命名图片（自定义名字+序号）
   d. 【Http下载】→ 下载图片到文件夹
6. 【新建Excel】→ 写入标题/正文/标签/图片路径
7. 【关闭保存】Excel
```

### 批量扩展思路
把链接放Excel → 读取Excel → 循环调用上面的流程

### 关键技巧
- XPath定位轮播图：小红书轮播图片用swiper-slide类，需加过滤防重复
- 自定义对话框让用户灵活配置输入
- 标签采集：用相似元素列表批量获取再拼接

## 案例三：吴（GitHub szwnba）— 小红书自动发布机器人（小白教程）

**来源**：GitHub Issue（2024-06）
**地址**：https://github.com/szwnba/affweb/issues/575

### 业务场景
从准备好的素材列表 → 自动打开小红书创作者中心 → 上传图片/填标题/填正文/填标签 → 发布

### 流程步骤
```
1. 【打开网页】→ https://creator.xiaohongshu.com/publish/publish
   （提前登录好创作者平台）
2. 点击「上传图文」按钮 → 进入图文发布页
3. **上传首图**：
   - 【上传文件】→ 选中img_list的第一个元素路径
   - 操作目标指向上传区域元素
   - 等待1秒
4. **循环上传剩余图片**（ForEach列表循环）：
   - 判断 loop_inten_index > 0（索引>0表示非首图）
   - 【上传文件】→ 上传当前图片
   - 操作目标指向「+上传更多」按钮
   - 等待1秒
5. **填写帖子信息**：
   - 【填写输入框(web)】→ 填标题
   - 【填写输入框(web)】→ 填正文（先清空）
   - 【键盘输入】→ 回车换行（让标签在新行）
   - ForEach列表循环加标签 → 勾选「追加输入」
6. 点击「发布」
```

### 关键指令
- `上传文件` — 网页自动化，直接上传本地文件路径
- `ForEach列表循环` — 循环上传图片/加标签
- IF条件 — 判断循环索引>0才执行（跳过已上传的首图）
- 追加输入 — 标签输入时必须勾选，否则会覆盖前面的

### 注意
小红书发布页是先上传首图→再追加更多图片，所以需要区分

## 案例七：林焱RPA — 小红书云帆商品自动上架

**来源**：CSDN（2025-12）
**地址**：https://adg.csdn.net/695343155b9f5f31781c072b.html

### 业务场景
Excel商品数据 → 自动登录小红书商家后台 → 批量上传填充 → 图片处理 → 上架

### 核心流程
```
1. 读取商品数据Excel → 验证清洗数据
2. 打开小红书商家后台 → 登录
3. 逐行循环：
   - 填标题/描述/分类
   - 填价格/库存
   - 图片上传（File.Exists检查路径）
   - 设置规格属性
   - 提交 → 标记成功/失败到Excel
4. 分批次处理（10个一批）+ 3秒间隔防封
```

### 企业级最佳实践
- 数据清洗：过滤特殊字符
- 图片路径检查：File.Exists防报错
- 分批次+延时：避免触发平台反爬
- 日志记录：成功/失败标记到Excel

## 知识点总结

### 通用指令组合
| 场景 | 推荐指令 | 备注 |
|------|---------|------|
| 采集列表 | 批量抓取数据 | 一键抓取结构化数据 |
| 采集单页 | 获取元素对象 | 精确捕获元素内容 |
| 批量获取标签/图片 | 获取相似元素列表 + ForEach循环 | 配合相对定位 |
| 上传图片 | 上传文件 | 直接传本地路径 |
| 获取AI文案 | HTTP请求 (GET/POST) | 对接任意API |
| 用户输入配置 | 自定义对话框 | 灵活参数输入 |
| 定时执行 | 定时触发器 | Cron表达式 |

### 小红书操作注意事项
1. 首图→追加图片，顺序不能乱
2. 标签输入必须用「追加输入」模式
3. 正文填完要插入换行再写标签
4. 图片轮播用XPath：`//div[contains(@class,'swiper-slide')]//img`
5. 创作者发布页：https://creator.xiaohongshu.com/publish/publish

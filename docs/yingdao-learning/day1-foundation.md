# 影刀RPA 系统性学习 — 第1天：基础篇

## 一、影刀整体认知

### 影刀是什么
- 无代码可视化RPA工具，Windows/Mac/信创
- 通过拖拽积木搭建自动化流程
- 支持354+官方指令，分7大类
- 5.26+版本支持直接 `print()`，低版需 `from xbot import print`
- 有「魔法指令」功能：聊天式自动生成指令（基于大模型AI Coding）

### 影刀界面结构
1. **左侧**：指令面板（分类树形结构）
2. **中间**：流程编辑区（流程图形式）
3. **右侧**：流程+全局变量面板
4. **底部**：运行日志/输出面板
5. **顶部**：智能录制、数据抓取、影刀浏览器、运行、调试按钮

#### HTTP请求积木关键参数
- 方法：GET/POST/CONNECT/HEAD/PUT/DELETE/OPTIONS/TRACE/PATCH
- URL：请求地址
- 协议头：多行，`HeaderName:Value`
- 协议体：请求体
- 保存响应结果到：存为变量
- 响应结果包含：`status_code`, `content_type`, `content_encoding`, `content`

#### JSON配套指令
**转换为Json对象** + **从Json对象中提取值** = 完美替代你之前缺的JSON解析积木！

## 三、核心概念

### 变量
- 影刀有自己的变量系统，支持文本/数字/列表/字典/布尔/日期时间
- Python模式：变量名手动输入，不能从列表选
- 全局变量：`glv["变量名"] = 值`（Python代码段中使用）
- 流程参数：子流程之间传参
- 变量作用域：普通变量在流程内，全局变量跨流程

### 元素捕获
- 捕获单个元素：选中网页/桌面元素
- 捕获相似元素：获取一组相似元素
- 智能元素定位：AI分析网页结构，自动选最稳定的定位策略
- 元素锚点：用附近不变的静态文本锚定元素
- 如果元素定位失败：加等待/用智能定位/加重试

### 流程编排
- 流程从上到下顺序执行
- 模块化设计：拆分为多个子流程
- 异常处理：try-catch+恢复机制
- 日志记录：关键步骤加打印日志

### 魔法指令
- 直接描述需求生成指令（如："从页面提取店铺名称、粉丝数"）
- 支持智能修复：报错时一键修复
- 底层有大模型能力

## 四、Python代码段集成

### 基本用法
```
插入代码段(Python) → 在编辑框写Python代码
```

### 变量传递
- 代码段中的变量 = 影刀流程变量（同名即可）
- 全局变量：`glv["变量名"] = 值`
- print输出：新版直接 `print()`，旧版 `from xbot import print`
- 调用Python模块：新建py模块文件 → 调用模块指令 → 执行main函数

### 可用的第三方库
- 影刀自带Python环境（Anaconda）
- 可 `pip install` 安装任何包
- 支持 `urllib` / `requests` / `json` / `re` 等标准库

## 五、常见问题处理

### 元素定位失败
- 原因：页面加载慢/元素属性变化/动态内容
- 解决：加等待/智能定位/重试/更换定位策略

### Cookie获取不全
- Chrome升级后部分Cookie带partitionkey标记
- 方案1：降级浏览器到119以下
- 方案2：升级影刀到5.16+，用新指令带partitionkey筛选
- 方案3：用JS获取 `document.cookie`

### 执行慢
- 等待时间过长/网络延迟/电脑性能
- 优化：减少等待/关闭占用程序

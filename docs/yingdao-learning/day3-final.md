# 影刀RPA 系统性学习 — 第3天：收尾篇（登录态+定时执行+完整方案）

## 一、登录态保持方案

### 核心问题
小红书频繁掉登录，这是RPA自动化发布最大的坑。原因：
1. Cookie未持久化保存
2. 小红书反爬机制升级
3. 操作频率过高被判定为非人类行为
4. 设备指纹异常

### 方案1：浏览器用户数据目录（推荐，最简单）

影刀的「打开网页」指令 → 高级设置 → **用户数据目录**
- 填一个固定路径如 `C:\rpa_profile\xiaohongshu`
- 第一次手动扫码登录 → 登录状态自动保存到该目录
- 以后每次启动都复用这个目录，自动保持登录态

**操作步骤：**
```
1. 打开网页指令
2. 浏览器：Chrome
3. URL：https://creator.xiaohongshu.com
4. 高级设置 → 用户数据目录：C:\rpa_profile\xhs
5. 第一次运行时手动扫码登录
6. 之后每次运行自动保持登录态
```

### 方案2：Cookie导出/导入（账号池方案）
1. 第一次登录成功后，用「获取Cookie」指令保存到Excel
2. 下次启动用「设置Cookie」指令提前注入
3. Key指令：获取Cookie → 保存到Excel → 设置Cookie

### 方案3：Python代码段获取/设置Cookie
```python
# 获取当前网页的cookies并保存到文件
import json
cookies = driver.get_cookies()
with open("cookies.json", "w") as f:
    json.dump(cookies, f)

# 设置cookies恢复登录态
with open("cookies.json", "r") as f:
    cookies = json.load(f)
for cookie in cookies:
    driver.add_cookie(cookie)
```

### 方案4：扫码登录（首次）+ 用户数据目录（后续）
最稳的方案：**首次手动扫码 → 后续自动复用**

## 二、操作频率与反检测

### 关键原则：模仿人类
- 每次操作后加随机等待（0.5~3秒）
- 不要机械式高频操作
- 用「固定等待」+随机参数模拟人类节奏

### 推荐延迟配置
| 操作 | 延迟 |
|------|------|
| 打开网页后 | 等待页面完全加载（等待元素指令） |
| 点击后 | 1~2秒 |
| 填写完 | 0.5~1秒 |
| 发布前 | 2~3秒（模拟检查） |

## 三、异常处理策略

### 常见异常及处理
| 异常 | 原因 | 处理方式 |
|------|------|---------|
| 找不到元素 | 页面加载慢/元素未出现 | 加等待 → 重试 → 截图 |
| 登录失效 | Cookie过期 | 重新登录子流程 |
| 发布失败 | 网络/内容审核 | 记录日志 → 下一条 |
| 弹窗 | 验证码/提示 | 自动处理弹窗指令 |

### 异常处理流程
```
主要流程 → try-catch包裹
  ├─ 成功 → 继续下一步
  └─ 失败 → ①截图保存现场 → ②记录日志 → ③重试2次 → ④跳过当前任务
```

### 关键指令：自动处理弹窗
影刀有「自动处理弹窗」指令，可检测并自动关闭各种弹窗
（验证码/登录框/提醒窗等）

### 日志记录
- 关键步骤加「打印日志」指令
- 出错时截图保存（用「截图」指令）
- 记录到外部日志文件

## 四、定时执行方案

### 方式1：影刀内置定时触发器（个人版可用？）
- 客户端自带「定时触发」功能
- 支持时间表达式（Cron）
- 可设置：每天/每周/每月特定时间自动执行

### 方式2：Windows计划任务
- 用Windows系统自带的任务计划程序
- 定时启动影刀RPA的指定应用
- 命令行：`shadowbot.exe --run=应用ID`

### 方式3：企业版调度中心
- 需要企业版账号
- 控制台 → 任务管理 → 新建常规任务
- 支持Cron表达式
- 可监控运行状态和执行记录

### 推荐配置（每天发布1篇）
```
执行时间：每天10:00（上午高峰期前）
Cron: 0 10 * * *
```

## 五、完整方案总结

### 技术架构
```
AI生成文案（DeepSeek API）
       ↓
影刀RPA流程
  ├─ 打开小红书创作者中心（用户数据目录自动登录）
  ├─ HTTP请求 → 获取AI文案
  ├─ 转换为Json对象 → 解析文案
  ├─ 点击「发布笔记」
  ├─ 填写标题/正文
  ├─ 上传图片（拖拽文件到元素）
  ├─ 添加标签
  └─ 点击发布
       ↓
数据回传（HTTP请求→云服务器）
```

### 影刀流程搭建清单（12步）

| 序号 | 积木 | 参数 | 说明 |
|------|------|------|------|
| 1 | 打开网页 | URL: creator.xiaohongshu.com, 用户数据目录: C:\rpa_profile\xhs | 自动登录 |
| 2 | 等待元素 | 捕获登录后首页的特征元素 | 确保登录完成 |
| 3 | 点击元素(web) | 捕获「发布笔记」按钮 | 进入发布页 |
| 4 | 等待元素 | 等待发布页面加载完成 | 防页面未加载 |
| 5 | HTTP请求 | GET/POST → DeepSeek API | 获取AI文案 |
| 6 | 转换为Json对象 | 将响应内容转为Json | 解析文案 |
| 7 | 从Json对象中提取值 | 提取title/content/tags | 拆出字段 |
| 8 | 填写输入框(web) | 捕获标题输入框，填入title | 填标题 |
| 9 | 填写输入框(web) | 捕获正文编辑器，填入content | 填正文 |
| 10 | 拖拽文件到元素/系统对话框操作 | 上传图片 | 图片上传 |
| 11 | 填写输入框(web) | 标签输入框，填入tags | 加标签 |
| 12 | 点击元素(web) | 捕获「发布」按钮 | 发布 |

### 需要注意的坑
1. **第一次运行**：需要手动扫码登录一次
2. **图片上传**：小红书的上传组件可能是自定义UI，拖拽上传不行就切换到Windows对话框方案
3. **正文编辑器**：小红书用富文本编辑器，可能需要清空默认内容后再填入
4. **等待时间**：每个操作之间加1~3秒延迟，不要太快触发风控
5. **发布频率**：建议一天1~2篇，不要频繁发
6. **内容审核**：AI生成的文案如果太"机器人"，可能需要人工润色
